Example Google style docstrings. # module summary

# module description This module demonstrates documentation as specified by the Google Python Style Guide. Docstrings may extend over multiple lines. Sections are created with a section header and a colon followed by a block of indented text.

# section name: # section description # Example에는 함수, method, 모듈 등의 사용 예시를 작성 .. rubric:: 예제

Examples can be given using either the Example or Examples sections. Sections support any reStructuredText formatting, including literal blocks:: # :: <- html에서 코드 블록 생성을 위해 사용

$ python example_google.py

# section은 unindented text가 시작되거나 새 section이 시작되면 break Section breaks are created by resuming unindented text. Section breaks are also implicitly created anytime a new section starts.

# 만약 *args, **kwargs가 사용되는 경우 반드시 목록에 포함 If *args or **kwargs are accepted, they should be listed as *args and **kwargs.

The format for a parameter is:

# 이름 (자료형, optional): 설명, line은 모두 동일하게 indentation되어야 함
name (type): description
    The description may span multiple lines. Following
    lines should be indented. The "(type)" is optional.

    Multiple paragraphs are supported in parameter
    descriptions.

# Args는 html에서 Parameters로 자동 변경 :type param1: :param param1: The first parameter. :type param1: int :param #: obj: <- python 내부 keyword 표시?, 단어: consolas로 강조 :param # 두 번째 줄부터는 추가 indentation 필요: :type param2: :param param2: The second parameter. Defaults to None.

Second line of description should be indented.

  • **kwargs – Arbitrary keyword arguments.
